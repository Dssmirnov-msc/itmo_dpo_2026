# Постановка задачи ФОС

## Дисциплина

**Название:** Моделирование динамических систем

**Предметная область:** применение дифференциальных уравнений, численных методов и ИИ-инструментов для построения, анализа и интерпретации моделей динамических процессов.

## Цель разработки ФОС

Разработать фонд оценочных средств, позволяющий оценить, насколько обучающиеся умеют применять модели динамических систем для решения прикладных задач, обосновывать выбор модели, проводить вычислительные эксперименты, интерпретировать результаты и представлять их в инженерно-документационной форме.

ФОС должен быть связан с РПД и компетентностно-ролевой моделью КРМ 3.0, а также поддерживать последующую оценку проектных работ студентов.

## Видение курса

Курс проектируется как базовый обзорно-практический курс с прикладными кейсами. Он должен соединять математические основы динамического моделирования с задачами, релевантными ИИ-направлениям: анализ временных процессов, симуляция, прогнозирование, идентификация параметров, исследование устойчивости и объяснение поведения систем.

Продвинутый трек курса может быть полезен для роли ML Researcher, где важны математическая строгость, исследовательская постановка задач и разработка новых подходов.

Базовый трек ориентирован на ML Engineer, Data Analyst и AI CDO: для этих ролей важны прикладная постановка задачи, корректная интерпретация модели, понимание ограничений и способность использовать результаты моделирования при создании или оценке ИИ-решений.

## Целевые роли КРМ

| Роль | Ожидаемый уровень | Назначение роли в курсе |
|---|---:|---|
| ML Researcher (Исследователь МО) | продвинутый | исследование моделей, анализ устойчивости, постановка новых задач моделирования |
| ML Engineer (Инженер МО) | базовый | практическая реализация моделей, вычислительные эксперименты, интеграция с ML-пайплайнами |
| Data Analyst (Аналитик данных) | базовый | анализ динамических данных, визуализация, интерпретация результатов |
| AI CDO (Руководитель цифровой трансформации) | базовый | понимание ценности, ограничений и рисков применения динамического моделирования в ИИ-проектах |

## Предварительные оси оценивания

На этапе разработки ФОС предполагается использовать две основные оси оценки, согласованные с техническим заданием:

1. **Программное решение / вычислительный эксперимент**: корректность постановки модели, реализация, воспроизводимость, качество экспериментов, анализ результатов.
2. **Документация / инженерное обоснование**: ясность постановки задачи, связь модели с предметной областью, описание данных и ограничений, интерпретация результатов, качество представления выводов.

## Предварительный состав оценочных средств

Для дальнейшей разработки ФОС предполагаются следующие оценочные средства:

- практические задания по моделированию и анализу динамических систем;
- кейс с построением и исследованием модели для прикладной ситуации;
- проектная работа с вычислительным экспериментом;
- peer-review проектных решений;
- защита проекта перед комиссией;
- чек-лист самооценки студента.

## Требования к будущему ФОС

ФОС должен:

- отражать содержание РПД;
- связывать оценочные средства с компетенциями и индикаторами КРМ 3.0;
- использовать 10-балльную шкалу с понятными уровнями достижения;
- содержать рубрики для проектной работы, peer-review и защиты;
- быть воспроизводимым для преподавателя без дополнительных устных пояснений;
- поддерживать как базовый, так и продвинутый уровень освоения дисциплины.
