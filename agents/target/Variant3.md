# Вариант 3  
## Локализация источника аварийного загрязнения в городской ливневой сети

**Тип проекта:** командный научно-инженерный кейс повышенной сложности  
**Рекомендуемый состав команды:** 2–3 человека  
**Допустимый состав команды:** 2–4 человека  
**Виртуальный испытательный стенд:** EPA SWMM 5.2  
**Основной тип задачи:** обратная задача по ограниченным и зашумлённым измерениям

> Общие требования к проектной работе, составу отчёта, воспроизводимости, использованию ИИ-инструментов, защите и оцениванию определяются документом `Project_main.md`.

---

# 1. Задание

Разработать и проверить решение для локализации аварийного поступления загрязняющего вещества в городскую ливневую сеть.

После дождя датчики в нескольких контрольных узлах сети зарегистрировали загрязнение. Точное место аварии, время начала выброса и масса поступившего вещества неизвестны. Диспетчерская служба должна определить наиболее вероятный источник загрязнения и направить аварийную бригаду для отбора проб, локализации сброса и защиты водного объекта.

Необходимо:

1. по ограниченным измерениям определить наиболее вероятное место поступления загрязнения;
2. оценить время начала выброса;
3. оценить массу загрязняющего вещества;
4. сформировать ранжированный список возможных источников;
5. определить дополнительные точки отбора проб, позволяющие уточнить решение;
6. проверить устойчивость решения к шуму, пропускам и изменению дождевого события;
7. подготовить инженерную рекомендацию для аварийной службы.

Способ решения команда выбирает самостоятельно.

---

# 2. Описание инженерной ситуации

## 2.1. Объект

Рассматривается городская ливневая сеть, включающая:

- водосборные территории;
- дождеприёмные и смотровые колодцы;
- трубопроводы;
- накопительные сооружения;
- регулирующие элементы;
- один или несколько выпусков в водный объект.

В сети установлено ограниченное число датчиков. Они могут измерять:

- концентрацию загрязняющего вещества;
- расход;
- глубину потока;
- уровень воды.

Измерения поступают с дискретностью 5 минут.

## 2.2. Авария

В одной из заранее определённых потенциальных точек происходит однократный аварийный выброс условного растворимого загрязняющего вещества.

Для базового варианта принимается, что загрязнитель:

- полностью смешивается с потоком в узле поступления;
- не осаждается;
- не распадается в течение рассматриваемого периода;
- не вступает в химические реакции;
- переносится по сети вместе с водой.

Источником может быть промышленная площадка, парковка, складская территория, участок дорожной сети, технологический выпуск либо выбранный узел сети.

## 2.3. Решение аварийной службы

По результатам работы системы диспетчер должен получить:

1. наиболее вероятный источник;
2. не менее трёх альтернативных источников в порядке убывания соответствия данным;
3. оценку времени выброса;
4. оценку массы;
5. оценку надёжности вывода;
6. перечень точек для дополнительного отбора проб;
7. описание сценариев, которые невозможно различить по имеющимся датчикам.

---

# 3. Виртуальный испытательный стенд

Преподаватель предоставляет модель в формате `.inp`, подготовленную для EPA SWMM 5.2.

Рекомендуемый масштаб модели:

- 20–40 водосборов;
- 25–50 узлов;
- 30–60 трубопроводов;
- 1–3 накопительных сооружения;
- 1–3 выпуска;
- 8–15 потенциальных точек аварии;
- 4–6 контрольных датчиков качества воды.

Модель должна поддерживать:

- нестационарные осадки;
- гидравлический расчёт;
- перенос загрязняющего вещества;
- внешнее поступление загрязнителя в выбранный узел;
- сохранение временных рядов концентрации, расхода и уровня.

Для всех команд используется одна и та же базовая сеть. Параметры скрытых испытательных аварий не раскрываются.

---

# 4. Исходные данные

Команде предоставляются:

1. файл модели SWMM;
2. схема сети;
3. перечень потенциальных источников;
4. расположение контрольных датчиков;
5. набор известных сценариев для разработки;
6. временные ряды осадков;
7. временные ряды измерений датчиков;
8. описание единиц измерения;
9. правила формирования скрытого тестового набора.

Для известных сценариев могут быть доступны источник, время начала, длительность и масса выброса. Тестовые сценарии содержат только доступные диспетчеру измерения.

---

# 5. Параметры аварийных сценариев

Источник выбирается из множества:

$$S=\{s_1,s_2,\ldots,s_m\},$$

где $m$ — число потенциальных мест аварии.

Время начала аварии $t_0$ может находиться до начала дождя, во время роста интенсивности, вблизи пика либо на спаде стока.

Для обязательной части используется кратковременный выброс продолжительностью от 5 до 30 минут.

Масса загрязнителя $M$ изменяется в установленном преподавателем диапазоне, например:

$$0{,}5\leq M\leq5\ \text{кг}.$$

В испытания могут включаться:

- случайный шум концентрации;
- предел обнаружения;
- пропуски отдельных измерений;
- задержка передачи;
- отказ одного датчика;
- шум расхода и уровня.

---

# 6. Цель проекта

Необходимо восстановить параметры аварии:

$$\widehat{\theta}=(\widehat{s},\widehat{t}_0,\widehat{M}),$$

где:

- $\widehat{s}$ — оценка источника;
- $\widehat{t}_0$ — оценка времени начала;
- $\widehat{M}$ — оценка массы.

Помимо единственной оценки команда должна формировать ранжированный список источников:

$$R=(s_{(1)},s_{(2)},\ldots,s_{(k)}),$$

где для обязательной части $k=3$.

Инженерная цель состоит не только в минимизации численной ошибки, но и в выдаче аварийной службе решения, пригодного для организации обследования сети.

---

# 7. Ограничения задачи

## 7.1. Ограничение по времени принятия решения

Предварительное решение должно быть сформировано не позднее чем через 60 минут после первого обнаружения загрязнения.

Расчёт после поступления очередной порции данных должен занимать не более 60 секунд на компьютере, используемом при защите.

## 7.2. Ограничение на доступные данные

Запрещается использовать:

- концентрации в узлах без датчиков;
- истинный расход в неизмеряемых узлах;
- будущие измерения относительно момента принятия решения;
- истинные параметры скрытого сценария;
- служебные выходные файлы SWMM, недоступные диспетчеру.

## 7.3. Ограничение на структуру аварии

В обязательной части действует один источник, происходит один выброс и рассматривается один загрязнитель. Свойства вещества известны, топология сети не меняется.

## 7.4. Ограничение на дополнительный отбор проб

После предварительной локализации разрешается рекомендовать не более трёх дополнительных точек отбора проб.

---

# 8. Сценарии разработки и тестирования

Открытые сценарии должны охватывать разные источники, массы, моменты выброса, формы дождя и начальные условия сети.

Валидационный набор должен отличаться хотя бы одним существенным фактором: дождём, интенсивностью, временем выброса, уровнем шума или сочетанием источника и погодного события.

Рекомендуемый скрытый тестовый набор:

- не менее 20 сценариев;
- каждый потенциальный источник встречается хотя бы один раз;
- часть дождевых событий отсутствует при разработке;
- не менее 20% сценариев содержат пропуски либо отказ датчика.

---

# 9. Требуемый ход исследования

## 9.1. Анализ сети

Необходимо:

1. построить граф сети;
2. определить направление движения воды при разных расходах;
3. исследовать времена прохождения сигнала;
4. определить источники с близкими путями переноса;
5. выявить источники, неразличимые по текущей системе датчиков;
6. определить влияние дождя на концентрацию и разбавление.

## 9.2. Анализ измерений

Необходимо исследовать:

- время первого обнаружения;
- форму концентрационной кривой;
- максимальную концентрацию;
- интегральную нагрузку;
- задержку между датчиками;
- связь концентрации с расходом;
- чувствительность к пределу обнаружения.

## 9.3. Разработка решения

Команда самостоятельно определяет представление сети и аварии, способ оценки источника, времени и массы, способ выражения неопределённости и правило выбора дополнительных проб.

## 9.4. Проверка

Необходимо проверить известные и новые дождевые события, малые и крупные выбросы, шум, пропуски и отказ одного датчика.

---

# 10. Показатели качества

Доля сценариев, в которых истинный источник занял первое место:

$$Acc_1=\frac{N_{\mathrm{top1}}}{N}.$$

Доля сценариев, в которых истинный источник вошёл в первые три позиции:

$$Acc_3=\frac{N_{\mathrm{top3}}}{N}.$$

Ошибка времени:

$$MAE_t=\frac{1}{N}\sum_{i=1}^{N}|\widehat{t}_{0,i}-t_{0,i}|.$$

Ошибка массы:

$$MAPE_M=\frac{100\%}{N}\sum_{i=1}^{N}\left|\frac{\widehat{M}_i-M_i}{M_i}\right|.$$

Дополнительно допускается использовать число рёбер, гидравлическое расстояние или время переноса между истинным и выбранным источником.

---

# 11. Минимальные критерии работоспособности

На скрытом тестовом наборе решение должно стремиться обеспечить:

- $Acc_3\geq0{,}80$;
- медианную абсолютную ошибку времени не более 20 минут;
- медианную относительную ошибку массы не более 35%;
- время формирования решения не более 60 секунд;
- явное предупреждение, если данных недостаточно для надёжной локализации.

Невыполнение отдельного целевого значения не означает автоматическую незачётность, если команда показала пределы идентифицируемости и предложила обоснованные меры улучшения.

# 13. Содержание инженерной рекомендации

Рекомендация должна отвечать на вопросы:

1. Где наиболее вероятно произошла авария?
2. Какие ещё источники нельзя исключить?
3. Когда начался выброс?
4. Какова вероятная масса вещества?
5. Какие датчики подтверждают вывод?
6. Где следует взять дополнительные пробы?
7. Какие участки сети обследовать первыми?
8. Требуется ли временно перекрыть выпуск?
9. Насколько надёжен вывод?
10. Какие данные необходимы для уточнения?
11. В каких сценариях система может ошибиться?
12. Какие датчики следует добавить при модернизации?

# 14. Дополнительная часть

Допускается исследовать два одновременных источника, вещество с распадом, изменение топологии из-за засора, активный выбор следующей точки измерения, оптимизацию системы датчиков и последовательное уточнение решения.

---

# 15. Недопустимые упрощения и ошибки

Не допускается:

1. использовать истинные параметры скрытого сценария;
2. использовать будущие измерения;
3. оценивать качество только на сценариях, совпадающих с обучающими;
4. исключать сложные источники из итоговой оценки;
5. считать концентрацию независимой от расхода;
6. скрывать неоднозначные результаты;
7. всегда выдавать один источник без оценки надёжности;
8. оценивать массу только по максимальной концентрации без проверки;
9. объявлять решение системой реального аварийного реагирования без натурной валидации.
