# Инструкции проекта

**Цель проекта:** Данный проект посвящен разработке рабочей программы дисциплины (РПД) и комплексного фонда оценочных средств (ФОС) по профильным ИИ-дисциплинам

Полное ТЗ на разработку - source/ТЗ_Разработка_ФОС_репозиторий.docx

## Видение курса

Название - "Моделирование динамических систем".

Содержание - правила и принципы применения дифференциальных уравнений для моделирования в различных ситуациях и доменах знаний с применением инструментов и техник ИИ.

Роли, для которых курс может быть полезен:

Продвинутый уровень: ML Researcher (Исследователь МО)

Базовый уровень: ML Engineer (Инженер МО), Data Analyst (Аналитик данных), AI CDO (Руководитель цифровой трансформации).

Команда пока рассматривает как более приоритетную базисную реализацию - как обзорный курс с интересными практическими кейсами.

## Ключевые этапы работы

Работа состоит из нескольких этапов:

-   Этап 0 (инфраструктурный): развертывание основных папок и артефактов, подготовка шаблонов
    -   создать пространство на файлообменнике (выполняет пользователь)
    -   создать git-репозиторий (выполняет пользователь)
    -   создать требуемые папки на основе ТЗ (выполняет пользователь)
    -   заполнить установочные файлы: README.md, FOS_BRIEF.md, LICENSE
-   Этап 1 (подготовительный): разработка РПД
    -   Сформировать цель и задачи дисциплины
    -   Создать библиотеку образцов РПД (выполняет пользователь)
    -   Создать тематический план курса
    -   Определить пререквизиты и связь с другими дисциплинами по КРМ
    -   Сформировать компетентностную модель на основе КРМ 3.0
    -   Создать систему оценки результатов обучения (краткое описание, которое ссылается на развернутый ФОС, создаваемый на этапе 2)
-   Этап 2 (результирующий): разработка ФОС
-   Этап 3 (презентационный): публичная защита результатов проектной работы

## Правила

-   четко исполняй указания пользователя
-   пользователь задает конкретную последовательность работ, в данном документе прописан только верхний уровень
-   действуй на основе материалов указанных в source
-   не меняй никаких материалов в папке source
-   временные и промежуточные файлы клади в temp
-   если в результате работы создается какой-то код, или повторяющийся промт - то клади его в scripts
-   рисунки, которые будут использоваться в *.md файлах таргета должны будут быть положены в папку images
-   результаты работы пиши в папку target
-   инструкции, отчеты, справочные и сопроводительные материалы пиши в папку docs
-   в системе не установлен git, пользователь будет делать коммиты в ручном режиме, поэтому исключи git из своего расписания, но вместо этого веди файл commits.txt в папке docs
