# 2. Постановка задачи лабораторной работы

## 2.1. Прикладная постановка

В задачах моделирования динамических систем часто требуется аппроксимировать не отдельное числовое значение, а отображение, которое переводит одну функцию в другую: например, правую часть уравнения в решение, сигнал управления в траекторию системы или поле коэффициентов в поле решения.

Лабораторная работа рассматривает учебную постановку операторного обучения. Вместо сложной физической системы используются синтетические функции одной переменной и простые операторы, для которых можно получить точные значения. Это позволяет сфокусироваться на архитектуре DeepONet, процедуре обучения и проверке инвариантности к дискретизации.

## 2.2. Исходные функции

Входные функции $f(x)$ определены на отрезке $[0,1]$ и задаются усечённым тригонометрическим рядом:

$$ f(x)=a_0+\sum_{k=1}^{5}\bigl(a_k\cos(2\pi kx)+b_k\sin(2\pi kx)\bigr). $$

Коэффициенты $a_k$ и $b_k$ генерируются случайно, например из нормального распределения $\mathcal{N}(0,1)$.

Базовая сетка обучения содержит $N=100$ точек.

Рекомендуемый объём данных:

-   обучающая выборка — 500 функций;
-   тестовая выборка — 100 функций.

## 2.3. Аппроксимируемые операторы

В работе необходимо аппроксимировать три оператора.

**Оператор интегрирования:**

$$ (T_1f)(x)=\int_0^x f(t)\,dt. $$

**Оператор дифференцирования:**

$$ (T_2f)(x)=\frac{df}{dx}(x). $$

**Оператор возведения в квадрат:**

$$ (T_3f)(x)=f^2(x). $$

Для оператора дифференцирования точное значение производной вычисляется аналитически:

$$ f'(x)=\sum_{k=1}^{5}2\pi k\bigl(-a_k\sin(2\pi kx)+b_k\cos(2\pi kx)\bigr). $$

Оператор интегрирования может быть вычислен численно кумулятивным методом, например методом трапеций. Оператор возведения в квадрат вычисляется поэлементно.

## 2.4. Операторная постановка DeepONet

DeepONet аппроксимирует операторное отображение $T:f\mapsto Tf$.

Архитектура включает две подсети:

-   **Branch-сеть** принимает дискретизированную входную функцию $f$ и формирует вектор признаков $\mathbf{b}\in\mathbb{R}^p$;
-   **Trunk-сеть** принимает координату $y$ и формирует вектор признаков $\mathbf{t}(y)\in\mathbb{R}^p$.

Значение оператора в точке $y$ вычисляется как скалярное произведение:

$$ (Tf)(y)\approx\sum_{i=1}^{p}b_i\,t_i(y). $$

Такая структура позволяет обучаться на фиксированной сетке, а затем вычислять значения результата в произвольных точках, поскольку Trunk-сеть определена для координаты $y$.

## 2.5. Инвариантность к дискретизации

После обучения на сетке из 100 точек модель применяется к тестовым функциям на сетках другого разрешения:

-   50 точек;
-   100 точек;
-   200 точек.

Для входа Branch-сети функция приводится к обучающей сетке, а координаты новой сетки подаются в Trunk-сеть. Если ошибка при изменении разрешения не возрастает существенно, это рассматривается как экспериментальное подтверждение инвариантности к дискретизации.

## 2.6. Требуемые результаты

В результате выполнения работы должны быть получены:

-   три обученные модели DeepONet: для интегрирования, дифференцирования и возведения функции в квадрат;
-   графики функции потерь в процессе обучения;
-   таблица относительных $L_2$-ошибок на тестовой выборке;
-   сравнение ошибок на сетках 50, 100 и 200 точек;
-   визуализации точных и предсказанных значений операторов;
-   выводы о качестве аппроксимации и влиянии дискретизации.

# 3. Описание хода лабораторной работы

## 3.1. Подготовка среды

1.  Подготовить среду Python 3.10+.

3.  Проверить доступность CPU или GPU для обучения модели.
4.  Зафиксировать random seed для воспроизводимости.

## 3.2. Генерация входных функций

1.  Создать равномерную сетку $x\in[0,1]$ из 100 точек.
2.  Для каждой функции сгенерировать коэффициенты $a_0,a_1,\dots,a_5,b_1,\dots,b_5$.
3.  Вычислить значения $f(x)$ на сетке.
4.  Сформировать обучающую и тестовую выборки.

## 3.3. Вычисление точных значений операторов

Для каждой функции из выборки:

1.  Вычислить интеграл $T_1f$ кумулятивным численным методом.
2.  Вычислить производную $T_2f$ аналитически по коэффициентам ряда.
3.  Вычислить квадрат $T_3f$ поэлементно.
4.  Проверить размерности входных и выходных массивов.

## 3.4. Формирование обучающих пар

Для каждого оператора сформировать набор примеров, где одна запись содержит:

-   дискретизированную входную функцию $f$;
-   координату $y$;
-   целевое значение $(Tf)(y)$.

Для каждой входной функции создаются пары со всеми точками сетки. Такой формат соответствует стандартному обучению DeepONet на парах "функция - точка - значение".

## 3.5. Реализация DeepONet

Создать три класса:

-   `BranchNet` — полносвязная сеть с входом размером 100 и выходом размером $p$;
-   `TrunkNet` — полносвязная сеть с входом размером 1 и выходом размером $p$;
-   `DeepONet` — модель, объединяющая Branch-сеть и Trunk-сеть через скалярное произведение признаков.

Рекомендуемые параметры:

-   размер скрытого слоя — 128 нейронов;
-   размерность признакового пространства — $p=50$;
-   функция активации — ReLU;
-   оптимизатор — Adam;
-   функция потерь — MSE.

## 3.6. Обучение моделей

Для каждого оператора:

1.  Создать отдельную модель DeepONet.
2.  Сформировать загрузчик данных.
3.  Обучить модель в течение 100-200 эпох.
4.  Сохранять историю значения функции потерь.
5.  При необходимости использовать часть тестовой выборки как валидационную.
6.  Сохранить обученную модель или параметры, достаточные для повторного запуска.

## 3.7. Оценка качества

1.  Получить предсказания модели для тестовых функций на основной сетке из 100 точек.
2.  Для каждой функции рассчитать относительную $L_2$-ошибку:

$$ \varepsilon=\frac{\|\widehat{Tf}-Tf\|_2}{\|Tf\|_2}. $$

3.  Усреднить ошибку по тестовой выборке.
4.  Сравнить качество для трёх операторов.
5.  Построить графики точного и предсказанного результата для 3-5 тестовых функций.

## 3.8. Эксперимент с изменением сетки

1.  Интерполировать тестовые функции на сетки из 50 и 200 точек.
2.  Подать в Branch-сеть значения функции, приведённые к обучающей сетке.
3.  Подать в Trunk-сеть координаты новой сетки.
4.  Получить предсказания на новых сетках.
5.  Вычислить относительные $L_2$-ошибки для сеток 50, 100 и 200 точек.
6.  Сделать вывод о наличии или отсутствии инвариантности к дискретизации.

## 3.9. Ожидаемые результаты

Ожидается, что:

-   для линейных операторов интегрирования и дифференцирования DeepONet даст высокую точность;
-   для нелинейного оператора возведения в квадрат ошибка может быть выше;
-   при изменении сеточного разрешения ошибка не должна значительно возрастать;
-   графики должны демонстрировать близость предсказаний к точным значениям операторов.

-   исследовать влияние размерности признакового пространства $p$;
-   добавить новый оператор, например сглаживание или свёртку с заданным ядром.
