# 2. Постановка задачи лабораторной работы

## 2.1. Прикладная постановка

В задачах моделирования динамических систем и процессов в сплошных средах часто требуется многократно решать PDE при разных полях параметров. Классический численный солвер решает уравнение заново для каждого нового поля коэффициентов, что может быть вычислительно дорого.

Нейронный оператор обучается по набору численных решений и затем аппроксимирует отображение от коэффициента уравнения к решению. В данной лабораторной работе эта идея изучается на примере уравнения Дарси.

## 2.2. Уравнение Дарси

Рассматривается стационарное уравнение диффузии на единичном квадрате $\Omega=[0,1]^2$ с однородными граничными условиями Дирихле:

$$ -\nabla\cdot\bigl(a(x)\nabla u(x)\bigr)=f(x), \quad x\in\Omega. $$

$$ u(x)=0, \quad x\in\partial\Omega. $$

Здесь:

-   $a(x)$ - поле проницаемости, положительная функция и коэффициент уравнения;
-   $f(x)$ - источник, для простоты $f(x)\equiv1$;
-   $u(x)$ - искомое поле давления.

## 2.3. Операторная постановка

Требуется аппроксимировать оператор:

$$ \mathcal{G}:a(x)\mapsto u(x). $$

Оператор каждому полю проницаемости $a(x)$ ставит в соответствие решение $u(x)$ уравнения Дарси.

Вход модели - поле $a(x)$ на равномерной сетке $N\times N$.

Выход модели - поле решения $u(x)$ на той же сетке.

## 2.4. Генерация полей проницаемости

Поля проницаемости задаются на равномерной сетке. Генерация выполняется по схеме:

1.  создаётся случайное поле $z(x)$ с заданной корреляционной структурой, например фильтрацией белого шума гауссовым ядром;
2.  применяется экспоненциальное преобразование для обеспечения положительности поля:

$$ a(x)=\exp(z(x)). $$

Поле $z(x)$ можно рассматривать как реализацию гауссовского процесса:

$$ z(x)\sim\mathcal{GP}(0,K). $$

Рекомендуемые параметры:

-   область $\Omega=[0,1]^2$;
-   сетка для обучения $N=64$, то есть $64\times64$ точек;
-   обучающая выборка - 1000 полей;
-   тестовая выборка - 200 полей;
-   длина корреляции $\ell=0{,}2$;
-   амплитуда $\sigma=1{,}0$.

## 2.5. Получение эталонного решения

Для каждого поля $a(x)$ необходимо решить уравнение Дарси численным методом.

Рекомендуемый подход:

1.  дискретизировать оператор $-\nabla\cdot(a\nabla u)$ на сетке;
2.  получить систему линейных уравнений:

$$ Au=f. $$

3.  решить систему итерационным методом, например методом сопряжённых градиентов;
4.  сохранить поле решения $u(x)$ как эталон для обучения и тестирования.

Рекомендуемые массивы данных:

-   `a_train` - тензор размером `[1000,1,64,64]`;
-   `u_train` - тензор размером `[1000,1,64,64]`;
-   `a_test` - тензор тестовых полей;
-   `u_test` - тензор тестовых эталонных решений.

## 2.6. Fourier Neural Operator

FNO аппроксимирует операторное отображение между функциональными пространствами. Основная идея архитектуры состоит в том, что интегральное ядро параметризуется в частотной области.

Общая схема FNO:

1.  входное поле $a(x)$ преобразуется в пространство признаков через линейный слой;
2.  последовательно применяются Fourier-слои;
3.  в Fourier-слое выполняется FFT, обучаемое преобразование младших частот и обратное IFFT;
4.  добавляется локальная линейная или нелинейная трансформация;
5.  финальный проекционный слой выдаёт поле решения $u(x)$.

Для функции $v(x)$ Fourier-слой можно записать как:

$$ v_{t+1}(x)=\sigma\left(Wv_t(x)+\mathcal{F}^{-1}\bigl(R\cdot\mathcal{F}(v_t)\bigr)(x)\right). $$

Где:

-   $\mathcal{F}$ и $\mathcal{F}^{-1}$ - прямое и обратное преобразования Фурье;
-   $R$ - обучаемая комплексная матрица, параметризующая интегральное ядро в частотной области;
-   $W$ - локальный линейный оператор, например свёртка $1\times1$;
-   $\sigma$ - функция активации, например GeLU.

## 2.7. Требуемые результаты

В результате выполнения работы должны быть получены:

# 3. Описание хода лабораторной работы

## 3.1. Подготовка среды

1.  Подготовить среду Python 3.10+.
2.  Подключить библиотеки NumPy, Matplotlib, SciPy и PyTorch.

4.  Проверить доступность CPU или GPU для обучения модели.
5.  Зафиксировать random seed для воспроизводимости.

## 3.2. Генерация данных

1.  Создать двумерную равномерную сетку `x = np.linspace(0, 1, N)`, `y = np.linspace(0, 1, N)`.
2.  Для каждого примера сгенерировать случайное поле $z(x)$ с заданной корреляционной структурой.
3.  Получить положительное поле проницаемости $a(x)=\exp(z(x))$.
4.  Сформировать обучающую выборку из 1000 полей и тестовую выборку из 200 полей.
5.  Проверить размерности массивов и корректность значений поля проницаемости.

## 3.3. Численное решение уравнения Дарси

Для каждого поля проницаемости:

1.  дискретизировать оператор $-\nabla\cdot(a\nabla u)$ методом конечных разностей;
2.  задать источник $f(x)\equiv1$;
3.  учесть однородные граничные условия Дирихле;
4.  решить систему линейных уравнений методом сопряжённых градиентов или другим выбранным солвером;
5.  сохранить полученное поле $u(x)$ как эталонное решение.

## 3.4. Реализация FNO

Допускаются два варианта реализации:

-   использование готовой модели `FNO2d` из библиотеки `neuraloperator`;
-   собственная реализация Fourier-слоёв на PyTorch.

Рекомендуемые параметры модели:

-   число удерживаемых Fourier-мод по каждому измерению - `(12, 12)`;
-   число скрытых каналов - 32;
-   число входных каналов - 1;
-   число выходных каналов - 1;
-   оптимизатор - Adam;
-   скорость обучения - `1e-3`;
-   функция потерь - MSE.

## 3.5. Обучение модели

1.  Сформировать `TensorDataset(a_train, u_train)`.
2.  Создать `DataLoader` с размером батча 32.
3.  Обучить модель в течение 100 эпох.
4.  На каждой эпохе вычислять предсказание `u_pred = model(a_batch)`.
5.  Вычислять MSE между предсказанным и эталонным решением.
6.  Выполнять обратное распространение ошибки и шаг оптимизатора.
7.  Сохранять историю функции потерь.

## 3.6. Оценка качества на тестовой выборке

1.  Перевести модель в режим оценки.
2.  Для каждого тестового батча получить предсказание $u_{\mathrm{pred}}$.
3.  Для каждого примера рассчитать относительную $L_2$-ошибку:

$$ \varepsilon=\frac{\|u_{\mathrm{pred}}-u_{\mathrm{test}}\|_2}{\|u_{\mathrm{test}}\|_2}. $$

4.  Рассчитать среднее значение и стандартное отклонение ошибки по тестовой выборке.
5.  Представить результаты в таблице.

## 3.7. Визуализация результатов

Для нескольких случайных тестовых примеров построить:

-   поле проницаемости $a(x)$;
-   эталонное решение $u(x)$;
-   предсказанное решение $\widehat{u}(x)$;
-   абсолютную ошибку $|u(x)-\widehat{u}(x)|$.

Визуализации должны позволять оценить, насколько хорошо модель воспроизводит пространственную структуру решения.

## 3.8. Эксперимент с инвариантностью к дискретизации

Цель эксперимента - проверить, что FNO, обученный на сетке $64\times64$, может делать предсказания на сетках другого разрешения без переобучения.

# 7. Методические рекомендации для преподавателя

## 7.1. Подготовка задания

Перед занятием рекомендуется проверить работоспособность среды, наличие библиотек NumPy, SciPy, Matplotlib, PyTorch и возможность установки `neuraloperator`.

## 7.2. Методические рекомендации для студентов

Следует начинать с небольшой выборки и проверки численного решения уравнения Дарси. После проверки структуры данных можно переходить к обучению FNO на полном наборе.

## 7.3. Вычислительная сложность

При ограниченных вычислительных ресурсах можно уменьшить число обучающих полей, число эпох или размер сетки. В отчёте необходимо явно указать, какие параметры были изменены.

## 7.4. Файлы варианта

Рекомендуемый минимальный набор:

-   Jupyter Notebook с кодом и отчётом;
-   файл с сохранёнными метриками или таблицами;
-   при необходимости файл с параметрами обученной модели.

## 7.5. Интерпретация результатов

Особое внимание следует уделить анализу пространственной структуры ошибки, сравнению качества на разных сетках и интерпретации скорости инференса FNO относительно традиционного солвера.

## 7.6. Упрощённая версия

В упрощённой версии допускается уменьшить размер сетки, число обучающих примеров или использовать готовый численный солвер без детального разбора его реализации.

## 7.7. Усложнённая версия

В усложнённой версии можно:

-   увеличить число обучающих полей;
-   исследовать влияние числа Fourier-мод;
-   исследовать влияние числа слоёв и скрытых каналов;
-   сравнить FNO с DeepONet или обычной CNN-моделью;
-   использовать более сложные поля источника $f(x)$;
-   рассмотреть другие граничные условия.
